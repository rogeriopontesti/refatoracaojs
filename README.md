## 📖 Sobre o projeto

Este projeto foi desenvolvido com o objetivo de demonstrar, na prática, a diferença entre um código considerado “amador” e um código mais limpo, organizado e profissional através do processo de refatoração utilizando JavaScript puro.

Durante o vídeo, mostramos como um sistema simples de cadastro de contatos pode rapidamente se tornar difícil de manter quando toda a lógica fica concentrada em uma única função. A partir disso, iniciamos uma refatoração completa separando responsabilidades, criando funções específicas e tornando o código mais escalável, reutilizável e fácil de entender.

O principal objetivo deste conteúdo é ajudar programadores iniciantes a entender que programar não significa apenas “fazer funcionar”, mas também criar códigos simples, organizados e preparados para manutenção futura.

---

## 🚀 O que é refatoração?

Refatoração é o processo de reorganizar e melhorar a estrutura interna de um código sem alterar o comportamento final da aplicação.

Na prática, refatorar significa:

- Melhorar legibilidade
- Separar responsabilidades
- Facilitar manutenção
- Reduzir complexidade
- Melhorar escalabilidade
- Evitar repetição de código
- Tornar o projeto mais profissional
